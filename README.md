# Aqua Manga

一个“公开只读、本机编辑、手动发布”的静态漫画与音乐收藏网站。

## 两种访问模式

### 公开浏览

正常访问网站地址，例如：

```text
https://你的用户名.github.io/仓库名/
```

访客只能搜索、查看漫画和播放音乐。页面不会显示添加、删除和发布按钮。

### 管理模式

在网站地址后添加 `?manage=1`：

```text
https://你的用户名.github.io/仓库名/?manage=1
```

管理模式会显示“添加收藏”，可上传漫画、截图、简介和音乐。

管理模式不是在线账号系统。其他人即使知道这个地址，也只能在他们自己的浏览器里制作草稿，无法修改你的 GitHub 仓库或公开网站。

## 发布新收藏

1. 用你平时管理网站的浏览器打开 `网站地址/?manage=1`。
2. 点击右上角“添加收藏”。
3. 选择“添加漫画”或“上传音乐”，填写资料并选择文件。
4. 上传后先在页面中检查封面、详情和播放效果。
5. 再次打开“添加收藏”，选择“生成发布包”。
6. 点击“下载 aqua-manga-publish.zip”。
7. 解压 ZIP，将其中的 `data`、`assets` 文件夹复制到本项目根目录。
8. 允许覆盖同名文件，然后把项目提交并推送到 GitHub。
9. 等待 GitHub Pages 部署完成，普通网站地址就会显示新内容。

## 删除收藏

1. 打开 `网站地址/?manage=1`。
2. 点击漫画卡片或音乐右侧的删除按钮，也可以在“管理收藏”中删除。
3. 删除只会先记录为本地草稿。
4. 重新生成发布包、覆盖仓库并推送 GitHub 后，公开网站才会完成删除。

旧图片或音乐可能仍留在 `assets` 文件夹中，但已经不会在网站中显示，可以之后手动清理。

## 公开数据结构

```text
data/content.json       漫画、作者、简介和音乐信息
assets/manga/           漫画封面及截图
assets/music/           音乐和音乐封面
```

访客看到的是这些已提交到 GitHub 的文件，不是你浏览器里的 IndexedDB 草稿。

## 文件限制

- GitHub 不允许提交超过 100 MB 的单个文件。
- 仓库不适合存放大量高码率音乐，建议上传压缩后的 MP3。
- 请确保你拥有公开分享相关漫画截图和音乐文件的权利。
- 发布前不要清除管理浏览器的站点数据，否则尚未导出的草稿会丢失。
